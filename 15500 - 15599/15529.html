Reported by jiasij...@gmail.com, Aug 3, 2014 
我是Android开发人员，经常要使用Android SDK Manager升级SDK，由于Google被封，祸及Developer,不通过goagent几乎是无法正常升级，因此，非常感谢goagent作者！但近几个版本包括目前最新的3.1.20都出现如下问题，在Android SDK Manager设置代理后，在Android SDK Manager Log 弹窗中出现两条红色错误信息并造成升级失败：
（1）Failed to fetch URL https://dl-ssl.google.com/android/repository/addon_list-2.xml,reason:peer not authenticated
（2）Failed to fetch URL https://dl-ssl.google.com/android/repository/addon_list-10.xml,reason:SSLPeerUnverified peer not authenticated
即使我强制SDK Manager使用http协议也没有用。奇怪的是，当我用3.1.6的goagent版本就没有这种情况，能正常升级Android SDK Manager。
不知作者能不解决下这个问题，衷心感谢！

#12 uuu...@gmail.com 
1）goagent（3.2.3） 在proxy.ini中添加：
dl-ssl.google.com = withgae,nocrlf,noforcehttps,nofakehttps
以前的版本可能不需要withgae。
2）android sdk manager 在option中勾上force https to http

#17 zenith....@gmail.com 
看了看goagent的手册，改成下面这样目前是好用了：

dl-ssl.google.com = withgae,nocrlf,noforcehttps,nofakehttps
.google.com = google_hk,noforcehttps,nofakehttps

我理解就是不要再把http强制转成https，既然SDK manager里已经设置了force https to http
