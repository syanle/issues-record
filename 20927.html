gost进阶：设置加密方式和密码以及同时运行WS和SS

假定
projectname工程名是demo-project
username，注册时填写的username，就是username
完整的websocket地址为demo-project-username.c9.io
客户端为windows系统

最近gost翻墙效果不错

一、设置gost加密方式为rc4-md5，密码为password

1、服务器端操作
进入到cloud9的demo-project编辑页面
估计之前已经运行过./gost -ws
鼠标点击一下下方的bash - "Cloning...区域，使焦点落在该区域
键盘按ctrl+c，在服务器端结束之前已经运行gost
出现$提示符
运行
./gost -m rc4-md5 -p password -ws

在浏览器中输入projectname-username.c9.io测试，看是否显示Bad Request，如服务器端已成功运行gost

2、客户端操作
windows系统
运行命令提示符cmd.exe
进入到gost所在目录
运行
gost -L :8899 -S demo-project-username.c9.io -m rc4-md5 -p password -ws

也可以把上述命令保存为批处理文件(.bat)，放在gost目录下直接运行

这样gost就设置了加密方式和密码
gost支持多种加密方式，按上述顺序和格式设置即可


二、gost同时运行WS和SS
gost不能同时运行WS和SS，办法是运行两个gost服务器端

1、cloud9创建新工作区
cloud9打开默认工作区编辑页面，一般是
https://ide.c9.io/username/demo-project
复制这段url

在cloud9的dashboard页面点击 create new workspace 再点击 clone from url
出现对话框
把刚才复制的url粘帖进去
Workspace Privacy:选择Open and Discoverable
点击create
出现提示框，要求更改工作区名称，输入新工作区名称
稍等几秒钟，demo-project下面就出现了新工作区，点击start editing，即可编辑

2、现在cloud9有两个工作区，各自运行一个gost，一个 -ws，一个 -ss，即可
具体请参考
https://docs.google.com/document/d/1Wxwl4HrvIpie9xIRRMzasdW_HAuthBFjqL_hQCKBIfE

3、作SS模式运行的gost要使用与WS模式不同的端口，请更改gost参数L，比如
./gost -L :9999 -S demo-project-username.c9.io -ss
即把gost-ss服务器端的端口设置为9999
也可以更改gost-ws客户端的端口，同样是更改参数L，格式是一样的


三、删除cloud9工作区
dashboard页面
点击工作区名称
在start editing右侧靠近屏幕边缘处有delete
点击delete
出现对话框，输入delete，再点击delete

#22 qgt...@gmail.com 
现在冒充大神的还真不少，借贴补充一下怎么设置SS模式以及自定义加密，有些人根本就没试验成功就来误导人：
1，C9.IO端 username@demo-project1:~/workspace/gost_1.2_linux_amd64 $ nohup ./gost -sm rc4-md5 -sp password -ws &  在clone一个workspace后重新安装gost，然后运行。注意-sm指定SS模式的加密方式，-sp指定加密密码。不加就会采用默认的加密和密码。
2，电脑端运行gost -L :8890 -S demo-project1-username.c9.io -sm rc4-md5 -sp password -ws -ss    注意命令行的一致性。
3，安卓手机上的影梭设置： 手机与电脑需要在一个局域网中，影梭服务器地址设置为电脑的局域网地址， 服务器端口设8890，加密方式与密码注意与上面两个一致。 电脑端不能关闭。
