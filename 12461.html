#2 ablegao 
这个原因是操作系统引起的。可以先修改proxy.user.ini :
[dns]
enable=1
listen=127.0.0.1:8053

然后命令行配置iptables 或者ipfw :
ipfw add 100 fwd 127.0.0.1,8053 tcp from any to any 53 in
ipfw add 200 fwd 127.0.0.1,8053 udp from any to any 53 in

这样端口访问53的数据会转到你的8053端口下面。

#5 vest.apr...@gmail.com 
补充iptables规则:
iptables -t nat -A OUTPUT -o lo -p udp -d 127.0.0.1 --dport 53 -j DNAT --to-destination 127.0.0.1:8053

不需要tcp规则,因为其拒绝client到goagent的tcp连接.
