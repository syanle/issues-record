Issue 17598: 基于Google App Engine for Python SDK的2步验证上传服务器端方法

Reported by guanglin...@gmail.com, Sep 29, 2014 
按照Google App Engine SDK for Python的上传服务器端方法，总结了一个稳定可靠的上传方法，已使用年余。因初始配置复杂，适合熟悉计算机的人使用，对有编程经验的人没有难度。仅以Windows操作系统为例说明。

0. 请安装可爱的自由软件Notepad++，以使编辑和运行工作更加方便。下载地址：http://notepad-plus-plus.org/。若将全局字体调整为verdana则更养眼。

1. 安装Python SDK
下载：https://www.python.org/downloads/release/python-278/，默认安装位置：C:\Python27。若会修改环境变量，请在全局path后追加C:\Python27。

2. 安装Google App Engine SDK for Python（墙外）
下载：https://cloud.google.com/appengine/downloads，安装时建议指定为C:\Google\google_appengine\。

3. 准备goagent
建议创建目录 C:\Google\goagent，将不同的版本以子目录存于这个目录下，以3.2.0 Beta 3为例，C:\Google\goagent\goagent-3.2.0b3。

4. 准备上传脚本，命名为upload_app.bat，存放于C:\Google\google_appengine\，内容如下(注：appid = ... 为方便编辑后面的app.yaml文件。另请将YOUR_EMAIL_ACCOUNT替换为您的注册邮件账户名称，也可以不是gmail的，请自行修改）

@rem is commentary in bat file!
@rem appid = YOUR_APP_ID01|YOUR_APP_ID02|...

@rem Without the ２-step authentication
@rem C:\Python27\python.exe  .\appcfg.py update ..\goagent\goagent-3.2.0b3\server\gae

@rem rollback if the last update fails
@rem C:\Python27\python.exe .\appcfg.py rollback  --oauth2 --email=YOUR_EMAIL_ACCOUNT@gmail.com ..\goagent\goagent-3.2.0b3\server\gae

@rem upload/update the app one by one with the 2-step authentication
C:\Python27\python.exe  .\appcfg.py --oauth2 --email=YOUR_EMAIL_ACCOUNT@gmail.com update ..\goagent\goagent-3.2.0b3\server\gae

4. 用Notepad++编辑C:\Google\goagent\goagent-3.2.0b3\server\gae\app.yaml，加入第1个appid（请将YOUR_APP_ID01替换为你的真实appid。每次只能上传一个，传一个之前相应地修改一下appid）:

application: YOUR_APP_ID01

5. 借助vpngate或用Freegate作代理上传（二选一）
在打开upload_app.bat的状态下，从Notepad++的运行菜单下单击Open current dir cmd，即进入到了目录C:\Google\google_appengine\的命令行窗口。注意：无论选下列哪种方法，第一次运行upload_app.bat时都要启动2步验证，请按弹出网页操作。以后就不需要了，除非换了机器或重装了系统。

5.1 借助vpngate
vpngate是全局模式，在获得稳定连接后，在上述命令行窗口运行upload_app.bat。

5.2 借助Freegate作代理
设置代理。在C:\Google\google_appengine\ 目录下创建批处理文件set_proxy.bat，内容如下：

@rem Use an http proxy server(https://cloud.google.com/appengine/docs/python/tools/uploadinganapp#Python_Using_an_HTTP_proxy)
@rem Use Freegate as the proxy.
set HTTP_PROXY=http://127.0.0.1:8580
set HTTPS_PROXY=https://127.0.0.1:8580

运行Freegate，稳定后先在上述命令行窗口执行set_proxy.bat（设置临时环境变量，每批次的更新或上传工作只运行一次，除非你关闭了上述命令行窗口），然后再运行upload_app.bat。

4. 回退的处理
若出现问题需要回退时，修改upload_app.bat，将标有rollback的下面一行打开：
...
@rem rollback if the last update fails
C:\Python27\python.exe .\appcfg.py rollback  --oauth2 --email=xxx@gmail.com ..\goagent\goagent-3.2.0b3\server\gae
...

5. 总结
上述方法成功率极高，缺点是初始配置复杂及不支持批量上传。未来计划结合phus.lu的uploader代码实现批量上传功能。
