Reported by langf...@gmail.com, Feb 12, 2014 
[dns]
enable = 1
listen = 127.0.0.1:53
remote = 168.95.1.1|114.114.114.114|114.114.115.115|8.8.8.8|8.8.4.4
cachesize = 5000
timeout = 2

系统的dns修改为127.0.0.1之后，问题就来了。不运行goagent的时候是没法正常上网的，虽然说我的goagent是常开的，但是总感觉这样作很不完美甚至没有必要的。

如果要使用goagent的dnsproxy，都要修改系统的dns，直接修改系统的dns为168.95.1.1不就成了，至少不会出现不运行goagent上不去网的问题，为什么要修改成127.0.0.1。

不知道goagent的这个dnsproxy功能作用体现在哪里

#1 cf16...@gmail.com 
127.0.0.1  本机的意思，windows上没有测试过，但在Tomato路由上开启goagent的dnsproxy，CPU使用率飙高100%，所以我也没有开启这项功能，只是在Dnsmasq自定义设置上加入以下两条：
address=/youtube.com/ytimg.com/redirector.c.youtube.com/8.8.8.8
address=/c.youtube.com/111.111.111.111
在防火墙脚本中加入以下：
iptables -t nat -I PREROUTING  -i br0 -p tcp -d 111.111.111.111 --dport 80 -j DNAT --to-destination 192.168.1.1:8086
iptables -t nat -I PREROUTING  -i br0 -p tcp -d 111.111.111.111 --dport 443 -j DNAT --to-destination 192.168.1.1:8086

#2 wwqgtxx 
作用体现在如果你直接修改系统的dns，你在命令行执行
nslookup m.youtube.com
获得的结果还是被污染的
如果你修改成127.0.0.1，那么你获得的结果就是对的

不信你可以尽情的去尝试
