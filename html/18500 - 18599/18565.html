Reported by YNyou...@gmail.com, Nov 11, 2014 
1. 如果您使用的是 proxy.user.ini 请打开 proxy.user.ini，如果您使用的是 proxy.ini 则打开 proxy.ini

2. 找到 
   google_cn = 一些东西
   google_hk = 一些东西
   
   然后如果「一些东西」为 203.208 开头或者 58,120 开头的等相关的国内 IP
   删掉替换成原有的：
   google_cn = www.google.cn|www.g.cn
   google_hk = www.google.com|mail.google.com|www.google.com.hk|www.google.com.tw|www.l.google.com|www2.l.google.com|www3.l.google.com|www4.l.google.com|www5.l.google.com|mail.l.google.com|googleapis.l.google.com|googlecode.l.googleusercontent.com|maps.l.google.com|code.l.google.com|cert-test.sandbox.google.com

   或者利用 GoGo tester 等 IP 检测软件测出一些可用 IP 替换

3. 找到：
   [gae]
   enable = 1
   appid = goagent
   password =
   path = /_gh/
   mode = https
   ipv6 = 0
   sslversion = TLSv1
   window = 6
   cachesock = 1
   headfirst = 1
   keepalive = 0
   obfuscate = 0
   pagespeed = 0
   validate = 0
   transport = 0
   options =
   regions =
   maxsize = 2097152
   改为和我给出的示例一样的形式
   或者根据自己的情况更改一些参数，但是确定 pagespeed = 0 
   如果还有 403 提示尝试使 obfuscate = 0 再试

PS:部分 pagespeed IP 还是可用的，可以自己测试（添加单个 cn IP 测试），然后剔除不能用的 IP 之后应该就可用了，不过死亡速度很快，剔除不可用 IP 应该不能作为永久解决方案

另外 certificate 以及上传相关的错误请直接看主页上
