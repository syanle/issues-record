#3 zhongofs...@gmail.com 
在 local 目录下的 proxy.ini 中有profile = <site>一项(我使用的版本的<site>为google_hk)，我将<site>改为google_cn后连接速度就加快了，也更少出现"return None, try again"字样，可以试试，注意<site>应为此文件中某个像：

[google_ipv6]
mode = http
hosts = 2404:6800:8005::2f|2a00:1450:8006::30|2404:6800:8005::84
sites = .google.com|.googleusercontent.com|.googleapis.com|.google-analytics.com|.googlecode.com|.google.com.hk|.googlegroups.com
forcehttps = groups.google.com|code.google.com|mail.google.com|docs.google.com|profiles.google.com|developer.android.com
withgae = 
withdns = 

的Entry最上方被'['和']'包含的项

#4 fangzhuo...@gmail.com 
我在校内没账号时用goagent2.1.7翻墙，也出现了楼主说的情况。经过楼上几位的提醒，我发现问题就出在proxy.ini的设置中。
在proxy.ini中有[google_cn]、[google_hk]、[google_ipv6]三个标签，profile ＝〈site〉要根据实际情况填写三个标签中的一个：比如我这种情况，学校有IPv6线路，在不登陆学校IPv4账号的情况，可以上六维，可以上Google，那么我就应该写成profile ＝ google_ipv6
我用以上方法解决了问题，不过我太懂这个，有说错的地方请列位大侠多多指教。
