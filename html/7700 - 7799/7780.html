在proxy.py里第997行，原来是

                        fetchserver = random.choice(self.fetchservers)
                        app_code, code, response_headers, response_rfile = self.urlfetch(self.method, self.url, headers, self.payload, fetchserver, password=self.password)

但是self.urlfetch的时候可能会出现SSL干扰，于是就加了一个检测SSL Error然后重试的机制。把上面几行替换成这样：
                fetchNeeded = True
                while fetchNeeded:
                    try:
                        fetchserver = random.choice(self.fetchservers)
                        fetchNeeded = False
                        app_code, code, response_headers, response_rfile = self.urlfetch(self.method, self.url, headers, self.payload, fetchserver, password=self.password)
                    except ssl.SSLError:
                        logging.warning('SSL Error detected. Retrying...')
                        fetchNeeded = True

改完以后似乎没有再出现断流的现象。有人愿意的话可以下载附件替换一下试试。如果想自己改的话注意Python的逻辑控制是通过缩进的空格数来控制的，要小心修改。
