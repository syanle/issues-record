#2 m.jackie...@gmail.com 
你的server:段为什么出现两三次？每个段（即未缩进的）只能出现一次。

server:
    # 设为1或2，输出更详细的日志
    verbosity: 1
...
    # 重要
    interface: 192.168.0.2 # (eg...)
    interface: 127.0.0.1
...
    # 重要
    port: xx
...
    # 全部可以采用默认
    msg-cache-size: 128m
    msg-cache-slabs: 8
    rrset-cache-size: 256m
    rrset-cache-slabs: 8
    cache-min-ttl: 3600
    cache-max-ttl: 86400
...
    do-ip4: yes
    do-udp: yes
    do-tcp: yes
....
    # 关键
    access-control: x.x.x.x/24 allow
    access-control: y.y.y.y/24 allow
....
    # 重要
    directory: "..."
    logfile: "unbound.log"
    use-syslog: no
    log-queries: yes
....
    # dnssec部分
    root-hints: "/.../root.hints"
    # 关键部分，如果是vpn环境下，可以应用validator, 否则关闭（支持dnssec的服务器不多，而且干扰相当严重）
    module-config: "iterator"

forward-zone:
    name: "."
    forward-first: yes
    forward-addr: 8.8.8.8 # (支持dnssec, 可以在vpn环境下试试)



大部分选项可以先采用默认，然后慢慢加。一下加那边多选项，而且日志还不开，鬼知道出什么毛病
dnscrypt干扰相当严重，早就放弃了。现在在vpn环境下跑unbound
