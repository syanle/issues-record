我己在一台主机(操作系统Ubuntu,以下称主机A)上安装goagent,现在希望让同一网段的其它主机仅将网关设定成主机A就可使用goagent提供的服务.

因为并不是所有的程序能容易地为自身设定代理服务,我现在就遇到一个难题.作为Python的初级学员,在安装一个Python模块时发现这个模块需要从pytz.sourceforge.net下载一些包,而这个网址却能通过代理才能访问.
以下是执行"python setup.py install --user"指令时出现的提示:
Reading http://pytz.sourceforge.net
Download error on http://pytz.sourceforge.net: timed out -- Some packages may not be found!

如果有Python专家知道如何为其设定代理,也请给予指点,毕尽它是我最初想解决的问题.

只不过,这不仅仅是python会遇到的问题.比如在使用maven工具时,也会遇到要从一些必需使用代理才可访问的网址下载一些jar包,比如"jasperreports.sourceforge.net".因此还是想找到某种全面的解决方法.

过去有利用iptables将主机A设成NAT服务器的经历,其实质是执行下面的脚本:
iptables -F
iptables -t nat -A POSTROUTING -o eth0 --jump MASQUERADE
iptables -t filter -A FORWARD -s 192.168.1.0/24 --jump ACCEPT
iptables -t filter -A FORWARD -d 192.168.1.0/24 --jump ACCEPT
iptables -t filter -A FORWARD ! -s 192.168.1.0/24 --jump DROP
ehco 1 > /proc/sys/net/ipv4/ip_forward
当时主机A是双网卡,eth0是其连外网的网卡.现在我想知道是否可以通过修改第二行指令来实现其它将网关设为主机A的主机可使用goagent服务?

浏览其它issue时发现有提将goagent部署在路由上的方法,所以感觉我的设想也许能现实.但我个人对iptables指令并不太熟,上面的脚本是我抄来的.在此将我的问题提出,希望能得到专家的指点.

谢谢!
