Reported by xenocide...@gmail.com, Jul 23, 2011 
描述：windows2003系统
发现在管理员账号下goagent执行正常，速度很快，并支持论坛（表扬一下，比我以前用的GAP，wallproxy都强）。
本人为教育网用户，使用IPV6连接GAE，系统hosts文件添加了相应的ipv6地址映射。并在此前的wallproxy下执行良好。

问题是goagent一但在在受限账户（power user组）中执行，使用http模式访问网页出现错误urlopen error [Errno 10061]，但是改为https模式后，则无错正常执行。

此问题最奇怪的在于该错误只发生于受限账户（power user组）时，而是用管理员账户则无论是http模式还是https都无错正常执行，而其他配置都相同。

错误信息具体如下：

Error response
Error code 502.
Message: ['<urlopen error [Errno 10061] >', '<urlopen error [Errno 10061] >', '<urlopen error [Errno 10061] >'].
Error code explanation: 502 = Invalid responses from another server/proxy..

#1 xenocide...@gmail.com 
发现问题的所在了，由于系统没有重启。在管理员账号中对HOSTS的改变不知为何不能应用到受限账号中，解决方法就是重启一下就行了

还有貌似系统hosts文件中关于proxy.ini的设置无法调用，也就是说在proxy.ini中的[google]的
hosts = www.google.cn

并不会自动调用hosts文件中对 www.google.cn的解析
