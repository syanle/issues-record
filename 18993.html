#1 chengwei.yang.cn@gmail.com 
我碰到这个问题是这样的，由于我在本地启动了 goagent，所以我把本地代理加到 shell 环境变量（http_proxy, https_proxy）中了，下次启动 proxy.py 的时候，这个环境变量是生效的，所以导致 goagent 使用了自己的 proxy。

解决办法很简单，启动 proxy.py 前，首先取消 http_proxy, https_proxy 等 proxy 变量，例如： $ unset http_proxy

#2 oneredap...@gmail.com 
unexpected indent 表示缩进不正确。语法方面的问题。
你把self.connect_timeout = 6和self.proxy_password = proxy_password对齐，6后面再按一下回车键。像下面这样。

    self.proxy_password = proxy_password
    self.connect_timeout = 6
另外，proxylib.py 中使用空格作为缩进，所以self.connect_timeout前面的空白是空格哟。不然也会unexpected indent哟。

#12 lanse8...@gmail.com 
将proxy.ini中的[listen]一段的ip由127.0.0.1改为0.0.0.0，则提供了全局域网的网络共享，如果只想给自己共享该fq代理，username和password填上即可

[listen]

ip = 0.0.0.0 

port = 8087

username =

password =

visible = 0

debuginfo = 0

上面这个配置通过网络能查到，坑爹的是proxy.ini文件中的[proxy]一段的默认配置，默认是这样的：

[proxy]

enable = 0

autodetect = 1

host = 10.64.1.63

port = 8080

username =

password =

由于我局域网没有代理直接能上外网，所以看到默认配置是enable为0，自然觉得不会有问题，可是每次启动后，上网，命令窗口都是：

AttributeError("'ProxyNet2' object has no attribute 'connect_timeout'",),这样的提示，网页上显示的是502，然后页面也是这个内容。反复倒腾，实际就只要将autodetect改为0就好了。也是醉了。

[proxy]

enable = 0

autodetect = 0

host = 10.64.1.63

port = 8080

username =

password =
