Reported by scy...@gmail.com, Oct 7, 2014 
 

操作系统版本：WIN7
浏览器版本：Chrome + switchysharpe 
GoAgent 版本：3.1.25
GoAgent 日志： 见上传的图片

 
这个问题比较奇怪，不知道怎么描述比较清楚。

开始，我使用 server +local 都是3.1.19 。server 有密码，local 也有密码，同时配置里写 options=rc4 和password  . 一切正常.

1.接着，我更新server+local 为 3.1.25, 都有密码，配置一样。一启动local 就会出现错误，configParser.parsingError 见图片。

2.于是，我再更新 server + local 为3.2.0版本，重新上传，同样设置密码，配置一样，一启动local ，也会出现同样的错误 configParser.parsingError ，就是说，本地客户端启动不了，还没到出现 "Cannot set attribute" 错误。

所以，我把server 密码去掉，重新上传server ,然后把local配置的密码和options=rc4去掉，这时正常了，访问twitter 和纽约时报都正常。但是，久不久会跳出GOOGLE的403页面（还是405页面？忘了），提示说无法连 /_gh/之类的，这个忘了截图了。访问国内的网站时，出现的概率比较多。

3.为了使用密码验证，于是，我只好回退到旧版本。考虑到 3.2.0 版本是新的架构，所以我重新上传了 server 的 3.1.25 版本，如上所述，无法使用其对应的客户端 local 3.1.25 。这时，我使用版本3.1.19的local， 去连版本3.1.25的server ，同样设置了密码+options=rc4，一切正常。

这就有点奇怪了。
