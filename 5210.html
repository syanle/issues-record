本不打算写这个东西的，因为网上确实有很多类似的教程了。不过之前的一些回帖让某些来这里的朋友误解了，很抱歉。来这里也学了不少知识，也很感谢作者。现在把我使用proxifier的过程简单说一下，算不上教程吧，抛砖引玉。必须要声明的是，我自己现在已经不用这个东西了，其实不太熟悉。有错误的地方多包涵，
这个过程我只在 proxifier 3.15 汉化版 32位win7中文版， goagent2.05下测试过，不保证在其他环境下一定能成功。

 首先确保已联网且直接使用goagent 可以翻墙。
proxifier 是3.15中文版，具体地址 google 关键词 "proxifier 中文"

1. 运行 proxifier  依次选择 配置文件-》高级-》HTTP代理服务器 勾选 启用http代理服务器支持

2. 选择 配置文件-》代理服务器 添加 地址为 127.0.0.1 端口 8087 协议为 http ，这时点测试
结果如下，表示已配置完，测试代理已通过。

	地址:	127.0.0.1:8087
	协议:	HTTP
	验证:	否

[43:07] 测试已通过。

[43:07] 延迟 < 0 毫秒
[43:07] 测试已通过。
[43:07] 测试已完成。

点确定后 提示是否开启默认代理，选择开启的话，默认所有的程序都走代理。

3. 最后还有一步，那就是要让goagent本身直连出去，不能被proxifier代理了，
依次选择，配置文件-》代理规则-》添加新规则， 名称随意，比如goagent,应用程序，浏览添加 proxy.exe, goagent.exe，动作，选择direct，其他默认不变，保存。

配置完毕

这样配置后原本不能设置代理的程序也可以通过代理联网了，浏览器也不用设代理了，但是缺点是只支持http，对https无效。也可以选择只对部分指定的程序使用代理，其他程序默认直连。
