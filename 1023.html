Issue 1023: 有关RangeFetch与Android系统中的OpenCore的兼容性问题
3 people starred this issue and may be notified of changes. Back to list
Status:  Fixed
Owner:  ----
Closed:  Oct 2011
Type-Defect
Priority-Medium


Sign in to add a comment
 
Reported by max.c...@gmail.com, Oct 3, 2011 

Android系统的流媒体播放使用的是OpenCore。它有一个比较奇怪的特性，对于headers里面小写的“content-length”和“content-range”会识别错误，必须使用“Content-Length”和"Content-Range"的写法。

于是如果在Android系统上使用GoAgent将无法正常播放Youtube视频。正确的处理方式可以见WallProxy 1.04版本中的RangeFetch实现。

修改后rangeFetch中的headers处理部分，需要按照这样的顺序连续的写到OpenCore，否则会出现pipe broken：
<code>
        tmp_headers = ''
        tmp_headers += 'Content-Type: ' + data['headers']['content-type'] + '\r\n'
        tmp_headers += 'Connection: close\r\n'
        tmp_headers += 'Accept-Ranges: bytes\r\n'
        tmp_headers += 'Content-Length: ' + str(end-start+1) + '\r\n';
        #logging.info('%s' % tmp_headers)
        self.wfile.write('HTTP/1.0 %d %s\r\n%s\r\n' % (data['code'],'',tmp_headers))
</code>
