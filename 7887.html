#2 ColorShi...@gmail.com 
官方文档:
python 2.6.6
关于class httplib.HTTPResponse(sock[, debuglevel=0][, strict=0])
Class whose instances are returned upon successful connection. Not instantiated directly by user.
New in version 2.0.
http://docs.python.org/release/2.6.6/library/httplib.html#httplib.HTTPResponse

而官方文档 python 2.7.3 还是未更新,(可能是文档问题)
class httplib.HTTPResponse(sock, debuglevel=0, strict=0)
Class whose instances are returned upon successful connection. Not instantiated directly by user.
New in version 2.0.
http://docs.python.org/2.7/library/httplib.html#httplib.HTTPResponse

但是在 python 2.7.3 httplib Source Code中
def httplib.HTTPResponse.__init__	(	 	self,
 	sock,
 	debuglevel = 0,
 	strict = 0,
 	method = None,
 	buffering = False 
)		

因此如果是windows平台直接使用 proxy.exe是没有问题的,运行环境都打包了.
而使用非windows平台,或者直接运行proxy.py 如果 python 不是2.7版本的话肯定有问题.
升级到 2.7.3就可以解决这个问题了.
