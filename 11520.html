关于近日由于GOOGLE调整服务器而导致BW无法正常使用的一些解释
2013-11-08 02:57:38
GOOGLE服务器IP调整中，导致Go-agent等一批GAE代王里无法正常工作。受其影响，BW出现的问题及解决方案如下：

1、U2B和其他BANNNED网站连接速度太慢或返回403错误
解决方案：
打开BW，按住键盘的option + command两键，同时双击第一项Go-agent，将用记事本打开其配置文件，找到“[google_cn]”一段，将其下面的h^o*s￥ts IP地址改为 “74.125.235.130|74.125.235.131|74.125.235.132|74.125.235.133|74.125.235.134|74.125.235.135|74.125.235.136|74.125.235.137|74.125.235.138|74.125.235.139|74.125.235.140”，保存后退出，重新连接即可，打开速度提升明显，并无403错误出现。(注：连接方式依旧是 google_cn)




2、其他网站无法打开。
解决方案：由于GAE urlfetch服务目前很不稳定，建议重新申请并部署新的APP ID使用。步骤：先打开 https://appengine.google.com , 用GOOGLE邮箱账户登录后，再点击下方的 “You have XX applications remaining.” 链接，进行ID申请，然后将此ID加入BW的Go-agent对话框中上传服务器部署。

3、如果已用完所有十个ID，无法申请新ID了该怎么办？
解决方案：删除部分旧ID，申请并部署新的ID。删除旧ID方法：打开 https://appengine.google.com , 用GOOGLE邮箱账户登录后，在APPLICATIONS列表页点击需要删除的ID，进入详情页，再点击左侧菜单列表的 “Application Settings”，在 Administration 分支下，进入管理员页，滚动到下部，找到“Disable or Delete Application”项，点击“Disable application”，然后关闭即可，再返回application列表页，你就能看到此ID已被关闭，然后下面的“You have XX applications remaining.” 里将不再为0，即可申请新的ID了。










4、上传ID时出现403错误
终端错误代码如下：
Rolling back the update.
Error 403: --- begin server output ---
<!DOCTYPE html>
<html lang=en>
  <a href=//www.google.com/><img src=//www.google.com/images/errors/logo_sm.gif alt=Google></a>
  <p><b>403.</b> <ins>That’s an error.</ins>
  <p>Your client does not have permission to get URL <code>/api/appversion/clonefiles</code> from this server.  <ins>That’s all we know.</ins>
--- end server output ---

解决方案：
打开 BW 点击 h^o*s￥ts 相关操作 - 修改与更新系统 h^o*s￥ts
选中所有有关 GOOGLE 的项目行数，按 command + X 剪切到记事本，保存到桌面，然后删除，保存h^o*s￥ts后退出文本编辑。
重新上传 Goagent ID，应该无问题了。
上传成功后，记得把记事本里的GOOGLE项目恢复回h^o*s￥ts，否则可能无法正常打开google code平台。


另，GAE代王里恢复情况请大家耐心等待，并建议时常关注 Go-agent 网站公告 http://t.cn/hdZKhM 和问题反馈页面 http://t.cn/zO4cJ8O
