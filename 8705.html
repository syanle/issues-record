#4 HewigOvens 
关闭的工作方式是这样子的:
org.goagent.local.ios.stop的launchd job会去monitor /tmp/goagent/stop 这个文件; widget改动了这个文件的话就会触发这个launchd job, 然后就会执行关闭的脚本. 由于widget是运行在mobile权限下的, 所以 /tmp/goagent/stop 的属主要是mobile才可以修改并触发关闭脚本. 

一般来说这种问题通过重装goagent-ios 0.0.4-3能够解决, 手动做的话步骤是:

0.su root
1.launchctl unload /Library/LaunchDaemons/org.goagent.local.ios.stop.plist
2. mkdir -p /tmp/goagent
3. touch stop /tmp/goagent/stop
4. chown -R mobile /tmp/goagent
5. launchctl load /Library/LaunchDaemons/org.goagent.local.ios.stop.plist

正常情况下launchctl list | grep goagent应该是像这样的:

- 0 org.goagent.local.ios.stop
8485 - org.goagent.local.ios

出现1的情况很有可能是因为/tmp/goagent/stop不存在, 所以stop的launchd job就不起作用了(可能是postinst的脚本写的不完善导致)
