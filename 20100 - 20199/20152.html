】仅供参考
】首先关于autorange相关值的释义 很有必要了解一下：
# 线程数 对于endswith列表中文件类型，开启N个线程数同时下载
threads = 3
#分段下载一次最大下载量 每个线程每次下载maxsize的量
#maxsize = 1048576
#首次读写量 首次读取waitsize大小返回给浏览器
#waitsize = 524288
#后续读写量 然后每次都读bufsize大小的量并返回给浏览器
#bufsize = 8192

】一、Goagent作者的建议
phus.lu：
bufsize 一定不要超过 32k, 如果使用 默认 iplist 的话，就保持 8k 就可以了
二、某位网友的建议
如果 Youtube 视频采用 DASH 模式，则 bufsize 不必调高
如果 Youtube 视频采用 NON-DASH 模式（需要利用一些拓展、插件调整，否则默认 DASH）则可以调到 8k 以上，但是如 Phus.lu 所说，不要高于 32k
三、鄙见：不建议如很多网友说的看高清HD，将这些值设定得过高（相对默认设置）


】关于U2B视频 
一、默认强制调用HTML5播放器（在此之前是FLASH播放器）
HTML5-FLASH播放器 下载速度对比
Html5下载速度只有Flash的60%左右
见
http://www.faith.ga/3061.html
https://xijie.wordpress.com/2014/11/16/%E5%BC%BA%E5%88%B6%E4%BB%A5flash%E6%A0%BC%E5%BC%8F%E6%92%AD%E6%94%BE%E6%8F%90%E9%80%9Fyoutube%E8%A7%86%E9%A2%91/
二、默认开启DASH（Dynamic Adaptive Streaming over HTTP
动态自适应通过HTTP流媒体），Flash player、HTML5 player均适配DASH功能。
此功能简要概括为两点：
1、根据网速 自动选择视频码率
2、
当一个视频开始播放时只缓冲一小段，当播放进度接近时再缓冲一小段，禁用了Dash Playback之后，播放一个视频时就会一直缓冲到头

】】解决方案 思路仅供参考
本人是：关闭HTML5播放器（即开启FLASH播放器）+关闭DASH
两点好处：稳定、流畅
（HTML5下某些码率音视频不同步、无声；开启FLASH的主要目的在于在加载视频后一直缓冲到底；本人在IP稳定之下接近满速物理带宽，ps：本人对画质要求不高）
要实现关闭HTML5播放器+关闭DASH，chrome商店以下U2B视频扩展兼具两者及更多功能:youtube center（扩展文件很大、高神难用、早已停止更新）、Magic Actions for YouTube、SPOI Options等。
经安装试用、测试一定时间后，发现一个规律：
在关闭HTML5播放器后，首先调用的任然会是HTML5播放器，(因已关闭HTML5)调用失败之后会立即跳转FLASH播放器。
均出现如下问题：
在关闭HTML5播放器时（即：开启FLASH播放器） 任然使用的是HTML5播放器（很大概率）
在关闭HTML5播放器时（即：开启FLASH播放器） DASH能完美关闭 
在开启HTML5播放器时                        DASH时常不能如愿关闭（）

判断是鱼与熊掌不可兼得吧或者使用了GA等中转网络后（延迟）的因素，同一个扩展想要实现这2个功能多少不稳定。

此路不通、遂改变思路：
1个扩展只关闭HTML5播放器+1个扩展自定义U2B视频码率（等同关闭DASH功能）
居然完美解决！！
】】解决方案方案如下：
Disable Youtube HTML5 Player（ 其实就一个关闭HTML5播放器的JS代码 4OKB，有基础的网友也可照此自己写）
SmartVideo For YouTube（181KB，自定义视频码率+一直缓冲，相对上诉提及的扩展要简洁易用）
