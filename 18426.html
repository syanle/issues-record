Reported by www.leno...@gmail.com, Nov 4, 2014 
尤其是出现“The requested URL /_gh/ was not found on this server.”错误的朋友。

上传完成后可访问https://appid.appspot.com/_gh/和https://appid.appspot.com/2/来测试是否部署成功。
例如：
访问https://ikeymobi01.appspot.com/_gh/会看到如下提示：
GoAgent Python Server 3.2.0 works, deployed at 2014-10-13 12:52:06
Password: i******i
访问https://ikeymobi01.appspot.com/2/会看到如下提示：
GoAgent 服务端已经在 2014-10-13 12:52:06 升级到 3.2.0 版本, 请更新您的客户端。

3.2.0 上传过的就不用再传了，3.2.1和3.2.2都只更新了客户端，服务器端没改变。

另外3.2.x的proxy.ini中[gae]path = /_gh/，不是以前的[gae]path = /2/，注意在proxy.user.ini（如果有这个文件的话）中同步修改。

Reported by smzlz1...@gmail.com, Yesterday (18 hours ago)
如果出现/_gh/问题的使用这个方法,应该OK!

1.https://www.google.com/settings/security
在google安全设置里面 把帐户所授权限-不够安全的应用的访问权限 改成启用即可 不需要设置两步验证

2.使用goegent文件夹下"server"里的uploader.bat重新上传一遍你的ID.

3.打开goegnt.exe,期初软件本身会进行调整,大概过2分钟就可以正常浏览FB YOUTUBE等网站了!
