#7 guanglin...@gmail.com 
(1)将代理服务器的域名或IP地址及端口配置在Proxifier上。
(2）...\local\proxy.ini中：
...
[proxy]
enable = 0
autodetect = 0
...
（3）若Proxifier探测到python进程有无限循环，要选择禁用探测。
（4）若PC性能足够好（如CPU为4核或更高，8G以上内存），可尝试用https://www.virtualbox.org搭建一个虚拟机(网卡改为桥接）来运行proxifier+goagent，作为同一局域网内的共享服务， ...\local\proxy.ini中做如下变更
[listen]
;ip = 127.0.0.1
ip = 0.0.0.0
...
注意，相应的，在要使用共享服务的PC上，将Firfox的Foxyproxy中增加一个代理192.168.1.2:8087（假设这台虚机的IP地址为192.168.1.2)；Chrome的SwitchySharp中，将原127.0.0.1改为192.168.1.2。
（5）若按（4）配置了，Proxifier中就用默认的rules。

#10 guanglin...@gmail.com

keepalive = 1
（4）不知Proxifier的同类其它产品是否可供选择，见http://en.wikipedia.org/wiki/Comparison_of_proxifiers，以解决#9所述问题。

#11 zhch6...@gmail.com 
内网goagent+Proxifier代理，综合各位经验，经实践成功，总结如下：
(1)将代理服务器的域名或IP地址及端口配置在Proxifier上。
(2）...\local\proxy.ini中：
...
[proxy]
enable = 0
autodetect = 0
...
（3）若Proxifier探测到python进程有无限循环，要选择禁用探测。
（4）请调低proxy.ini中(如下）window的值试试，如改为8,6 4 或2；注意设置keepalive = 1：
[gae]
...
window = 2  （我2也不行，改的1）
keepalive = 1

2、3、4每一步都很重要啊
