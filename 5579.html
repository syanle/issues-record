这是关于 500 Server Error 的解决方法，如果 Phus 觉得有用，建议链接到 FAQ 中。
几乎每次我升级，都会遇到这个问题，除了访问 Google 网站没有问题外，其他的网站都会出现 500 错误，包括 youtube 等非 Google 域名的所有网站，同时访问 code.google.com 会出现排版错误。
一遍遍看 FAQ 之后也没找到解决方法，因为 FAQ 没有关于 500 错误的说明。
首先，这里指的 500 错误的具体说明是：
500 Server Error
Error: Server Error
The server encountered an error and could not complete your request.
If the problem persists, please report your problem and mention this error message and the query that caused it.
解决这个问题的方法有两个，第一个是最有效的，第二个可以应急。
方法一：重新运行 \server\uploader.bat 上传服务器端程序；如果问题没有解决，则请继续重新上传，如果你发现多次重新上传之后依然没有解决此问题，那么请重新下载 GoAgent 软件包，然后最好换一个时间段或者网络环境（比如使用其他代理）再重复前面的上传动作。我的两次 500 错误都是这样解决的。具体原因我也不清楚，可能是第一次下载的 GoAgent 包不完整，导致上传的不完整；也可能是网络原因，上传不完整。总之这样可以解决 500 错误的问题。
方法二：这个是用来救急的，只在你没有时间使用方法一时使用。如果你发现访问其他网站都是 500 错误，但是访问 google.com 是正常的，那么至少 goagent 现在是“勉强可用的”，你可以通过修改
[crlf]
enable = 1 // 设为 1
dns = 8.8.8.8
sites = .youtube.com|.ytimg.com | 加上你遇到 500 问题的域名，比如“.yahoo.com”
这样你再访问 yahoo.com “有时”就不会出现 500 错误了。当然如果这个方法无效，还请继续尝试用第一种方法解决。

#6 hsgho...@gmail.com 
我刚刚经历了同样的问题，现已解决，并且找到了我的症结所在。但解决方法并非如楼主所述仅靠反复上传。我遇到的问题在于现在的 Google App Engine 默认已经使用 High Replication Server，在开启这个的情况下，如果你曾经在上传过程中修改过 app.yaml 里的第二行 version 参数的话，它会在服务器上保存每次不同版本的程序，并且仅有一个版本设置为 defalut（默认是第一个上传的版本），即正在运行的版本。在这种情况下，你需要登录进 appengine.google.com，并且在 dashboard 中对你的程序手动设置 defalut 为新版本才可。不然新版本的 local 客户端和旧版本的 server 端是没办法正常配合使用的，只能保证最基本的对 google 的访问而无法访问其他网站的内容。
这是我遇到的问题，希望能给出现同样问题的朋友提供启示。
