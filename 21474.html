看了meek的介绍。。
https://trac.torproject.org/projects/tor/wiki/doc/meek
然后就想既然meek-server要部署到GAE上，那么goagent就能作为meek－client的前置代理了。。。。

1.下载
git clone https://git.torproject.org/pluggable-transports/meek.git

进入appengine，修改app.yaml第一行“application:YourAppID”

2.创建APP
这个不用再提了，不过现在创建APP可选两个集群，一个是美国的，一个是欧盟的，我两个都创建了一个。

3.部署
然后就是部署上传了。用过很多办法，都没有办法上传到GAE，最后是下载了这个东西GoogleCloudSDKInstaller.exe才成功的。。。
这里碰到很多问题。

首先、要修改GoogleCloudSDKInstaller.exe安装路径，不能有中文，不能有空格。PS：它自动安装的就有一个空格，如果不小心点了。安装后，要自己去修改路径把空格给去掉。（5555,自己在这里找了很多好久的问题）

其次、要修改它自动安装的python的编码问题。把python下lib中的mimetypes.py文件中的ctype = ctype.encode的三行注释掉。

然后就是用Google Cloud SDK Shell去上传了，最后，最好用goapp deploy上传，我用appcfg.py update上传失败。

找了第三方SDUpload，都传不上去。。

PS：希望大神能用goagent自带的上传工具写个教程啊。。。。。

4使用
这个就是最后一步了，在torrc文件中加入
HTTPSProxy 127.0.0.1:8087
UseBridges 1
Bridge meek 0.0.2.0:1 url=https://YourAppID.appspot.com/ front=www.google.com
ClientTransportPlugin meek exec ./PluggableTransports/meek-client --log meek-client.log

皆大欢喜了。

打完收工。。。。。

#4 scattere...@gmail.com 
meek gae 和 goagent 没关系
app.yaml 不用修改, 可用 goapp deploy -application new_appid 指定新的appid
上传 appid 需要访问 appengine.google.com , 可通过 vpn , 或在 hosts , 或在本地 dns server 中加入可用ip指向 appengine.google.com
tor 启动 meek gae, 需要访问 www.google.com , 再走vpn意义就不大了, 可在 hosts , 或在本地 dns server 中加入可用ip指向 www.google.com
meek gae 的服务端20150222后就没有再修改, 但现在速度的确有提升, 应是中转服务器 meek.bamsoftware.com 的带宽加大了
用 meek gae 追求 youtube 速度, 目前还是算了吧
